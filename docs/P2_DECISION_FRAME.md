# P2 결정 프레임 — 도달 전 사전 약속 (v3 WS-C, 2026-07-20 작성)

> 목적: P2 게이트 도달 순간의 **즉흥 결정 방지**. 옵션과 비용을 지금 계산해 두고,
> 도달 시 이 문서의 트리대로 보고 → **결정은 사용자 전권**.

## 1. 도달 판정 (자동 — v_gate_status)

- **P2 = 해소 30+ AND 대표 Brier < 0.20** (`v_brier_primary` — research_status='failed' 제외 표본)
- 보고 형식 의무: `Brier X.XXXX (n=NN, 제외 m건: failed)` — 제외 표본 수 상시 병기 (검토질문 #3 규약)
- 예상 도달: **~2027-01** (수확 캘린더 이행 시 12/31 해소 31~33건)

## 2. K회 실행 결정 트리 (도달 시 보고서에 이 표를 채워 제출)

| 조건 (도달 시점 대표 Brier) | 권고 옵션 | 비용 영향 |
|---|---|---|
| **< 0.18** | K=3을 **EXIT 직결 5질문에만** 활성 (`AI_FC_REASONING_RUNS=3`은 전역이므로 질문별 적용 배관 필요 — 활성 결정 시 소규모 코드 1건) | 리서치 공유 + 추론 2회 추가 = **+$0.4~0.8/질문, 월 +$3~4** ($20 내 수용 가능) |
| **0.18 ~ 0.20** | K=1 유지 + **원인 분석 우선** — Murphy 분해(REL vs RES)·도메인 분해·티어별 분해로 병목 규명 후 재보고 | $0 |
| 특정 도메인 > 0.22 (n≥10) | 해당 도메인 **시그널 차단** (기존 v_domain_skill 규약 자동 발동) — K 논의와 독립 | $0 |

부속 판단 재료 (도달 보고서에 병기):
- 벤치마크 쌍대: LLM vs ML vs 시장 누적 Brier (쌍대 n 명시 — n<10이면 "판정 유보" 명기)
- 섀도 extremized 가상 Brier vs 공식 (실보정은 해소 100+ 게이트 — 여기선 관찰만)
- 티어별: lite − standard 격차 (표본 10+부터 — +0.05 초과 시 lite 폐지 검토)

## 3. 봉인 유지 (P2 도달로 풀리지 않는 것)

- 실보정(isotonic/Platt) — 해소 100+ · 앙상블 가중 학습 — 해소 200+ · edge 시그널 — P3(50+/<0.18)
- 예산 상한 변경 — 사용자 전권 (시스템은 제안 금지)
