# 릴리스 자가검증 프로토콜 (RE-AUDIT-260715 §D — 릴리스마다 그대로 실행)

> 사용법: 릴리스(커밋 묶음·산출물 갱신) 후 아래 블록을 Claude Code/Chat에 그대로 붙여넣는다.
> 기준 문서: docs/KNOWN_LIMITS.md (이미 알려진 한계는 제외하고 새 약점을 찾는다).

```
너는 C:\workspace\ai-investing의 릴리스 자가감사자다. 이번 릴리스의 변경분과 산출물을
적대적으로 검사하고 결함 목록을 산출하라. 칭찬 금지. 확신 없는 지적은 [가설] 표기.
결함마다 심각도(치명/중대/경미) + 근거(파일·값·재현식)를 붙여라.
docs/KNOWN_LIMITS.md에 이미 있는 항목은 제외하고, 없는 약점을 우선하라.
검사 순서:
1. [확률 정합성 검산] 확률을 제시하는 모든 산출물(시나리오·예측·base rate)에 대해:
   합계 100% · 상호배타/전체포괄 · 단조성(부분사건 P ≤ 초사건 P — 예: P(종점>X) ≤ P(터치 X))
   · 터치-종가 분해 · 시장내재와의 괴리 기록 여부. 검산 섹션이 없는 확률 산출물은 그 자체가 결함.
2. [결정-문서 정합] docs/DECISIONS.md의 최신 결정이 하류 문서에 반영됐는가.
   폐기·강등·비활성화된 값(예: LPPL 보정 tc)이 라벨 없이 재등장하는지 전 문서 grep.
   서로 다른 시점의 결정 문구가 한 문서에 공존하는지 검사 (개정 포인터 확인).
3. [자동 산출물 침식] 재생성 파일(quant/ml/market/dotcom_*_auto.md)의 caveat·강등 상태가
   생성 코드에 내장되어 있는가. 문서만 고친 시정은 재실행 1회로 침식된다 — 코드 게이트 여부 확인.
4. [메타 파일 무결성] 캘리브레이션·게이트 표현에 영향을 주는 모든 메타(overrides류)가
   원장급 규율(append-only·행 해시·시점 불변식: 생성시각 < 해소시각)을 갖는가.
   게이트류 판정은 primary/all 양쪽 뷰에서 평가되는가 (한쪽 은폐 채널 검사).
5. [빈티지] base rate 파일별 수집일 vs 최신 예측일 괴리 7일+ 경고 발화 확인.
   이력 스냅샷과 현행 스냅샷의 라벨 구분, 서로 다른 구성물(예: 타이밍 분포 vs 회의별 확률)의
   동일 라벨 사용 여부.
6. [불변성] sync --check 종료코드 0 · forecasts/.hashes ↔ git 정합 · --force 사용 이력이
   있다면 사유가 CHANGELOG에 기록됐는가 · forecasts/·ledger 무수정 (git log -p로 표본 확인).
7. [신규 계층 렌즈] 이번 릴리스에 추가된 모델·데이터 계층 각각에 대해 문서화 여부 확인:
   (a) 추정 대상의 정의 (무엇의 확률/값인가) (b) 표본 실체(n, 자기상관, 생존편향)
   (c) 벡터/특징의 상호 상관과 유효 차원 (d) 산출물이 프롬프트에 주입될 때의 앵커링 경로
   (e) 자기 기각 조건 — 어떤 실측이 나오면 이 도구를 강등하는가 (사전 명시 없으면 결함).
8. [산출 형식] ① 게이트 위반 ② 통계 오류/과장 ③ 문서-산출물 불일치 ④ 추가 리스크의
   4분류로 결함 대장 작성. 감사인 자신의 이전 지적이 반증되면 그대로 기록하라 —
   반증은 절차의 실패가 아니라 작동이다.
금지: forecasts/·ledger 수정, 확률 임의 재배정, 게이트·센티널 무단 조정, 결측 보간,
과거 질문 LLM 테스트. 판단이 필요한 항목은 시정하지 말고 결정 대기로 넘겨라.
```

운영 주기: 자가검증 = 릴리스마다 · 외부 감사(무접근 패키지 방식) = 분기 1회 또는 Phase 전환 시.
