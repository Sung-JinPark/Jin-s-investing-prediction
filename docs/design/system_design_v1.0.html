01

원문 해부 — 슈퍼포캐스터 AI의 작동 원리

CORE
AI 슈퍼포캐스터 = 프론티어 LLM + 스캐폴드(리서치 과정을 유도하는 프로그램: 프롬프트 체계, 도구, 서브에이전트 배치 규칙).

기사가 공개한 FutureSearch의 실제 작동 과정이 설계의 출발점이다. 감기 종식 프로젝트의 성공 확률 질문에 대해: 2분 만에 서브에이전트 3개 배치, 웹사이트 16개 리딩 → 5분에 최종 확률 7% 산출, 212개 출처 인용, 비용 $8. 다른 AI(Preseen)는 8.8%, 인간 슈퍼포캐스터는 5~10% — 세 예측이 수렴했다.

가장 중요한 단서는 AI의 추론 구조다. 답변이 "모든 조건이 동시에 성공해야 하는 conjunctive chain"으로 구성됐다: ① 생물학적 난이도(base rate: 50년간 백신 실패) ② 타임라인 제약 ③ 채택률 불확실성 ④ 측정 불가능성. 각 조건의 확률을 곱해 낮은 최종 확률에 도달하는 분해(decomposition) 기법 — 테틀록 슈퍼포캐스팅의 핵심을 그대로 구현한 것이다.

#기법기사 속 증거주식시장 적용
1정밀 질문화기한·임계값 있는 해소가능 질문 요구"NVDA 상승?"→"8/26 실적 DC매출 컨센 상회 확률"
2스캐폴드 오케스트레이션서브에이전트 3개, 16사이트/2분실적·공시·매크로·수급 병렬 리서치
3Outside view 우선"50년 백신 실패" base rate 먼저실적 서프라이즈율, 사이클 통계, 계절성
4Conjunctive 분해4개 조건 동시성공 확률 곱시나리오 조건 트리 (금리×실적×수급)
5대량 출처 근거화212개 출처 인용출처 그레이딩 (공시>IB리서치>언론>SNS)
6앙상블 수렴 체크7% vs 8.8% vs 5~10% 비교멀티모델·멀티런 중앙값 집계
7캘리브레이션 벤치마크Metaculus Elo·Brier, 월 0.9pt 개선전 예측 DB화 → Brier 추적
8인과 조건부 예측각주1: 조건부 시장의 함정 지적"인상 시 조정폭" 류 인과 시나리오
9속도·폭의 우위주당 수백 개 시장 자동 스캔이벤트 캘린더 전수 스캔·자동 재예측
10용량 인식"쉬운 돈은 소진된다" 인정edge 임계치 + 자금 상한 설계
💡 스캐폴딩의 가치 — 정량 근거
Metaculus 분석: 예측 특화 스캐폴드는 베이스 모델 9개월치 발전과 동등한 성능 향상. 즉 같은 Claude라도 스캐폴드를 어떻게 짜느냐가 9개월치 모델 격차를 만든다. 이 설계서의 존재 이유다. AI는 월 0.9 Elo씩 개선 중이며, 1년 내 최고 인간을 능가하는 추세선 위에 있다.
02

증거의 경계 — 검증된 것 vs 주장인 것

HONESTY

설계 전에 기사 증거의 경계를 정직하게 긋는다. 이것이 시스템 목표 설정을 좌우한다.

주장증거 수준비고
이벤트 확률 예측력 = 인간 톱과 동률✓ 검증Metaculus Cup 3위(Preseen), 통계적 동률
금융 질문에서 AI 우위✓ 검증Market Pulse(금융 전용 대회) 봇 1위 — 전 인간 제침
예측시장 수익 ($35→$2M)✓ 사례단, 본인이 "용량 소진·재현 불가" 인정
주식시장 +25% 초과수익? 주장1개 업체(FutureSearch), Scott 본인도 "운일 수 있음" 명시
전 도메인 초인간화? 논쟁Kapoor&Narayanan: 예측은 irreducible error로 인간 수준이 한계라는 반론 존재
🚨 설계상 결정적 함의
검증된 능력은 "주가 맞히기"가 아니라 "이벤트 확률화"다. 주식 가격은 무한한 연속 변수의 집합이지만, 가격을 움직이는 개별 이벤트(실적 비트, FOMC 결정, 제품 출시, S-1 제출)는 예측시장형 질문으로 변환 가능하며 그것이 정확히 AI가 인간 톱과 동률로 검증된 영역이다. 또한 주식시장은 Kalshi류 예측시장보다 훨씬 효율적이라 쉬운 alpha가 적다 — 기대치를 여기에 맞춘다.
✅ 따라서 시스템 목표의 서열
1차(검증된 영역): 나의 EXIT 트리거 이벤트를 확률화 — "MU 마진 반전 확률", "OpenAI S-1 제출 시점", "VIX 25+ 확률". 의사결정의 질을 즉시 높인다. 2차(조건부 확장): AI 확률 vs 시장내재확률의 괴리(edge) 검출 — 단 캘리브레이션 증명(Brier<0.18, 50문항+) 후에만. 3차(선택): 마켓뉴트럴 페어 아이디어 — FutureSearch 방식, 가장 후순위.
03

설계 철학 — 이벤트 우선, 가격은 파생

DESIGN
"주가를 예측하지 말고, 주가를 움직이는 이벤트를 예측하라. 그리고 시장이 그 이벤트에 매긴 가격과 내 확률의 차이가 임계치를 넘을 때만 움직여라."
원칙내용기사 근거
① 이벤트 → 가격모든 예측 대상을 해소가능 이벤트 질문으로 변환예측시장형 질문이 검증 영역
② Edge 없으면 무행동|AI확률 − 시장내재확률| > 임계치일 때만 시그널"쉬운 돈 소진" — edge는 희소 자원
③ 캘리브레이션이 왕예측을 안 맞히는 시스템은 즉시 발견되도록 전수 기록Brier/Elo가 유일한 진실
④ 자동화된 폭사람이 못 하는 것: 매주 수백 질문 스캔·재예측AI의 속도·성실성 우위
⑤ 라이브 검증 onlyLLM은 과거 결과를 학습에 내포 → 백테스트 무효사전지식 오염 문제
⚠️ 원칙 ⑤가 특히 중요하다
전통 퀀트는 백테스트로 검증하지만, LLM은 훈련 데이터에 과거 사건의 결과를 이미 알고 있어 과거 질문 백테스트가 원천적으로 오염된다. 따라서 이 시스템의 검증은 오직 미래형 라이브 포워드 테스트로만 가능하다 — 예측을 먼저 기록하고, 시간이 해소해 주기를 기다린다. 이것이 Phase 설계에서 "검증 기간"이 필수인 이유다.
04

시스템 아키텍처 — 7 레이어

DESIGN
L1
질문 엔진 (Question Engine)
포트폴리오 전략·이벤트 캘린더 → 해소가능 예측질문 자동 생성. 기한·임계값·판정기준 검증. 애매하면 재작성 후 확인 (기사의 프롬프트 원칙)
▼
L2
리서치 스캐폴드 (Research Scaffold)
오케스트레이터가 서브에이전트 N개 병렬 배치 — 실적/공시 에이전트, 매크로 에이전트, 수급 에이전트, 반대증거 에이전트. 웹검색·뉴스·SEC·FRED·실적캘린더 도구. 출처 그레이딩(공시>IB>언론>SNS)
▼
L3
추론 코어 (Reasoning Core)
① Outside view: base rate 라이브러리 조회 → ② Inside view 보정: 이번 사례의 차이점 → ③ Conjunctive/disjunctive 분해 → ④ Premortem(반대증거 3개) → ⑤ 확률 1% 단위 + 신뢰구간
▼
L4
앙상블 레이어 (Ensemble)
모델 다양화(Claude·GPT·Gemini) × 프롬프트 변형 × K회 실행 → 중앙값/절사평균. 발산(σ>임계) 시 자동 재조사 트리거 — 기사의 7% vs 8.8% 수렴체크 자동화
▼
L5
엣지 검출기 (Edge Detector)
AI 확률 vs 시장내재확률 비교 — 옵션 IV 분포, Kalshi/Polymarket 가격, 애널리스트 컨센서스. |edge| > 10%p AND 앙상블 수렴 시에만 시그널 발행
▼
L6
사이징 & 액션 (Sizing & Action)
Fractional Kelly(Half-Kelly) 사이징 — 기존 나의 리스크 프레임과 동일. 상관·집중도 제약. 출력: 매매 시그널이 아니라 "확률 + edge + 권장 액션" 리포트
▼
L7
캘리브레이션 & 로그 (Calibration DB)
전 예측 영구 기록 → 해소 시 Brier/log score 산출 → 캘리브레이션 커브 → 도메인별 skill 추적 → 못 맞히는 도메인 자동 차단. 시스템의 심장
💡 레이어 간 피드백 루프
L7의 캘리브레이션 데이터가 L3의 base rate 라이브러리와 L5의 edge 임계치를 지속 갱신한다. 예: 시스템이 "실적 비트" 도메인에서 Brier 0.14(우수)인데 "지정학" 도메인에서 0.24(무능)라면 → 지정학 질문은 자동으로 시그널 발행 차단, 정보 제공만. 이것이 기사가 말한 "능력 범위(circle of competence)"의 기계적 구현이다.

모듈 상세 스펙

DESIGN
모듈입력출력핵심 구현 포인트
L1 질문 엔진전략 문서, 이벤트 캘린더, 보유 종목표준화된 질문 객체 {질문, 기한, 판정기준, 도메인}LLM이 질문 초안 → 판정기준 rules-lawyer 체크 → 사용자 승인
L2 스캐폴드질문 객체근거 패키지 {출처 리스트, 핵심 사실, 반대증거}서브에이전트 4종 병렬: 펀더멘털·매크로·수급·데블스애드버킷(반대증거 전담)
L3 추론 코어질문 + 근거 패키지확률 + 분해 트리 + 신뢰구간 + 근거 요약base rate 라이브러리(실적 서프라이즈율, 사이클 통계 등 사전 구축)
L4 앙상블동일 질문 K개 독립 실행 결과집계 확률(중앙값) + 발산도 σK=5~9. σ>8%p면 재조사. 모델 2종 이상
L5 엣지 검출AI 확률 + 시장 데이터edge 크기 + 방향 + 신뢰도옵션 IV→내재확률 변환(Breeden-Litzenberger 근사), Kalshi API
L6 사이징edge + 캘리브레이션 이력액션 리포트 (확률·edge·Half-Kelly 권장 규모)자동매매 아님 — 의사결정 보조 리포트가 기본. 실행은 사람
L7 캘리브레이션전 예측 + 해소 결과Brier, 캘리브레이션 커브, 도메인 skillPostgreSQL. 예측 시점 스냅샷 불변 보존(사후 수정 불가)

추론 코어 — 프롬프트 설계

IMPL

기사 말미의 단순 프롬프트("슈퍼포캐스터처럼 생각하고 확률로 답하라, 질문이 잘못됐으면 재작성 후 확인하라")를 전체 추론 체계로 확장한 시스템 프롬프트 v1.0:

# ── 슈퍼포캐스터 추론 코어 시스템 프롬프트 v1.0 ── 당신은 캘리브레이션이 검증된 슈퍼포캐스터입니다. 아래 절차를 엄격히 따르세요. [0. 질문 검증] - 기한·임계값·판정기준이 명확한가? 아니면 해소가능 형태로 재작성하고 진행 전 확인을 요청한다. - 조건부 질문이면: 상관관계가 아닌 인과적 조건부("X를 실행했을 때")로 해석함을 명시한다. [1. Outside View — 반드시 먼저] - 이 질문이 속한 참조 클래스(reference class)를 정의한다. - 유사 사례의 base rate를 3개 이상 수집한다. (예: "최근 20분기 NVDA 실적 비트율 85%") - base rate 를 초기 anchor 확률로 설정한다. [2. Inside View — 보정] - 이번 사례가 base rate 와 다른 점을 증거와 함께 열거한다. - 각 증거가 anchor 를 몇 %p 움직이는지 명시하며 조정한다. [3. 분해 — Conjunctive / Disjunctive] - 사건 성립에 필요한 조건들을 트리로 분해한다. - 조건 각각의 확률을 추정하고, AND면 곱·OR면 합으로 결합한다. - 결합 결과와 [2]의 조정 결과가 다르면 원인을 규명한다. [4. Premortem] - "이 예측이 크게 틀렸다"고 가정하고, 가장 그럴듯한 원인 3개를 적는다. - 각 원인을 반영해 최종 확률을 재조정한다. [5. 출력] - 최종 확률: 1% 단위 (예: 23%, "약 20~25%" 금지) - 80% 신뢰구간, 핵심 근거 3줄, 확률을 바꿀 수 있는 관찰가능 지표 2개 - 모든 사실 주장에 출처를 붙인다. 검증 못한 주장은 [미검증] 표기.
💡 데블스 애드버킷 서브에이전트
스캐폴드의 서브에이전트 중 하나는 반대증거 수집 전담으로 설계한다. LLM의 확증편향(질문 프레임에 동조하는 경향)을 구조적으로 상쇄하는 장치다. 기사의 FutureSearch 답변이 훌륭했던 이유도 "성공 스토리"가 아니라 실패 조건들(생물학·타임라인·채택·측정)을 중심으로 추론했기 때문이다.

# orchestrator.py — 질문 1개의 생애주기 (의사코드) async def forecast(raw_question: str) -> ForecastReport: # [L1] 질문 정밀화 — 해소가능 형태 검증 q = await question_engine.formalize(raw_question) if q.needs_clarification: return ask_user(q.draft) # [L2] 스캐폴드 — 서브에이전트 4종 병렬 (기사: 3 subagents, 16 sites) evidence = await asyncio.gather( agent_fundamental.research(q), # 실적·공시·가이던스 agent_macro.research(q), # 금리·CPI·유가 agent_flow.research(q), # 수급·옵션·포지셔닝 agent_devil.research(q), # ★ 반대증거 전담 ) evidence = grade_sources(evidence) # 공시>IB>언론>SNS # [L3+L4] 추론 × 앙상블 — K회 독립 실행 probs = await asyncio.gather(*[ reasoning_core.run(q, evidence, model=m, seed=k) for m in ["claude", "gpt"] for k in range(3) # K=6 ]) p_final = median(probs) if stdev(probs) > 0.08: # 발산 → 재조사 return await forecast_deep(q, evidence, probs) # [L5] 시장내재확률과 비교 p_market = await edge_detector.implied_prob(q) # 옵션IV·Kalshi·컨센 edge = p_final - p_market # [L6+L7] 리포트 생성 + 불변 기록 report = build_report(q, p_final, edge, evidence, sizing=half_kelly(edge) if abs(edge) > 0.10 else None) await calibration_db.record(report, immutable=True) return report # scheduler.py — 자동 재예측 (AI의 폭 우위 활용) for q in active_questions: if q.near_deadline() or news_monitor.material_change(q): forecast(q) # 주간 정기 + 이벤트 트리거 재예측

결론 & 다음 액션

NEXT
기사가 증명한 것은 "AI가 주식으로 돈 번다"가 아니라 "AI가 이벤트 확률화에서 인간 톱에 도달했고, 금융 질문에서 특히 강하다"는 것이다. 이 시스템은 그 검증된 능력을 나의 EXIT 의사결정에 직결시킨다.

이 설계의 차별점 3가지: ① 검증된 영역(이벤트 예측)에 1차 목표를 두고, 미검증 영역(주가 초과수익)은 캘리브레이션 게이트 뒤로 배치했다. ② 기존 리스크 프레임(Half-Kelly, VIX·드로다운 하드룰)과 충돌 없이 조기경보 레이어로 통합된다. ③ 자체 개발 역량으로 3~6주면 P1 MVP가 가능하고, 월 $200~500로 상시 운영된다.
