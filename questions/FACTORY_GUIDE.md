# 질문 팩토리 가이드 — 판별가능 단기 질문 등록 규약 (WS1, v2 라운드 2026-07-20)

> 목적: 게이트 병목(해소 표본 속도)을 풀되, **코인플립성 질문으로 Brier를 오염시키지 않는다.**
> 근거: 기해소 2건(spx-up 50%→Brier 0.25, soxx-up 46%→0.29)의 교훈 — 판별 여지 없는 질문은
> 표본을 늘려도 캘리브레이션 증명에 기여하지 못하고 게이트 통과만 늦춘다.

## 1. 등록 필터 (아래 중 하나 충족 필수 — 코드 강제)

- **(a) 판별 여지**: base rate 또는 시장내재확률이 **[35%, 65%] 밖** — 정보가 확률을 극단으로
  밀 수 있는 질문 (예: S&P 분기 EPS beat율 ~75~80%).
- **(b) 정보 우위 논거**: 컨센서스 집계와 대체 데이터의 괴리 등, 시스템이 시장보다 나은 판단을
  할 수 있다는 논거를 명시.

**집행**: `created: 2026-07-21` 이후 질문은 registry `notes`에 **`등록필터: (a) ...` 또는
`등록필터: (b) ...`** 마커+근거가 없으면 — `forecast` 프리플라이트가 **오류로 차단**,
`sync`가 W2 경고. (2026-07-20 이전 질문은 grandfather.) 수치의 진위 검증은 사람 몫 —
코드는 근거 기재를 강제할 뿐이다.

## 2. 질문 시리즈 3종 (재생성 규칙 포함)

### 시리즈 E — 실적 beat (domain: earnings, 해소 주기 = 실적 캘린더)
- 원형: "X사가 YYYY-MM-DD 발표하는 분기 실적에서 **EPS가 발표 직전(D-1) 컨센서스를 상회**할 확률은?"
- 판정: 발표 EPS(non-GAAP 우선, 회사 관례 기준) > D-1 컨센 스냅샷. 동률 NO. 스냅샷은 예측 파일에 고정.
- base rate: S&P500 분기 EPS beat율 ~75~80% (FactSet) — 필터 (a) 충족.
- **재생성**: 실적 발표·해소 후, 다음 분기 발표일이 확정되는 시점에 같은 원형으로 새 질문 등록
  (질문 id: `<ticker>-eps-beat-YYYYq#`). 발표일 미확정이면 등록 보류 (deadline null은 프리플라이트가 차단).
- cadence: `[{once: true}, {from: D-3, once: true}]` — **r1 1회 + D-3 재예측 1회** (비용 통제).

### 시리즈 M — 지표 서프라이즈 (domain: macro, 해소 주기 2~5주)
- 원형: "N월분 {CPI 헤드라인 YoY / NFP 최초치}가 임계 X를 {상회/하회}할 확률은?" — 발표일 확정·판정 기계적.
- 임계는 **컨센서스가 아니라 고정 수치** (판정기준 불변 — nfp-jul2026-below100k가 원형).
- 필터: base rate/시장내재가 [35,65] 밖이거나(예: "CPI 가속" — 최근 추세상 낮음) (b) 논거.
- 재생성: 해소 후 다음 발표분으로 재등록 (`cpi-<month>-accel` / `nfp-<month>-below100k`).

### 시리즈 V — 이벤트 발생 (domain: corporate-event/market-regime, rolling)
- 원형: 기존 rolling 질문 패턴 (vix-25-90d, btc-alt-mania-90d) — 윈도우 시작 스냅샷 필수.
- 필터 (a): 윈도우 내 발생 base rate가 극단인 것만 (예: 90일 VIX25 터치 — 저변동 레짐에서 ~20~30%).

## 3. 비용 규율 (질문당 · 월간) — 2026-07-20 개정: 월 상한 $20

- **월 상한 $20** (사용자 결정 2026-07-20, config.MONTHLY_BUDGET). 초과 시 프리플라이트 자동 차단.
- 단기 시리즈 질문: **리서치 에이전트 2개 고정** (4-agent 확장 금지).
- $20 체제의 산술: 질문당 $2.5~4 → **월 실행 가능 ~5~8회 예측**. 등록 28개 활성 질문을
  전부 cadence대로 돌릴 수 없다 — **우선순위 트리아지가 기본 모드**:
  1. **1순위**: 해소 임박 질문의 r1 (첫 예측 없이 만료 = 무예측 만료 방지 — cpi-jun 사례 재발 금지)
  2. **2순위**: EXIT 트리거 직결 질문 (fomc·soxx·vix·mu)
  3. **3순위**: D-3 재예측 — 1·2순위 소화 후 여유 있을 때만 (생략 시 r1이 최종 스냅샷 담당)
  4. 장주기 질문(openai 등)은 이벤트 트리거 시에만
- 매 실행 전 `python -m ai_fc report`의 이달 비용 확인. 상한 상향은 사용자만 결정.
- 참고: Claude Code 스킬 경로(/forecast)의 세션 토큰은 이 상한과 별도 회계 —
  단 동일한 "지시 시에만 실행" 규율 적용.

## 4. 기계 판정 초안 (auto-resolve draft)

- `python -m ai_fc resolve --draft` — 가격 임계형 질문(ml/mapping.QUESTION_MAPS 등재)의
  판정 초안(outcome·증빙값·소스·신뢰도)을 일괄 출력. **원장 무기록.**
- 확정은 반드시 사람이: `resolve <qid> --outcome yes|no --evidence <근거>` (기존 confirm 경로).
- 공시 수치형(EPS·CPI·NFP)은 초안 비대상 — 사람이 공식 발표를 확인해 판정.

## 5. 등록 체크리스트 (신규 질문마다)

1. 해소가능성: 기한·임계·판정출처 명확? 동률 처리 명시?
2. 등록 필터 (a)/(b) 충족 + `notes: "등록필터: ..."` 기재?
3. 스냅샷 필요 항목(`required_snapshots`) 정의? (컨센서스 등 — r1에서 확정)
4. schedule = `[{once: true}, {from: D-3, once: true}]` (단기 표준)?
5. 발표일 출처 확인 + `notes`에 "일정 D-30 재확인" 표기?

> 이 가이드는 v2 라운드 스펙 §3 WS1의 정본. 자동 질문 '생성'은 게이트 후 (§4 봉인) —
> 등록은 항상 사람이 한다. 모든 산출물은 참고 의견 (P3 게이트 전).
